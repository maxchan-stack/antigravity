# Vibe Coding 測試練習系統 - 使用手冊

歡迎使用 **Vibe Coding 測試練習系統**！這是一個專門用來練習軟體測試、體驗不同 API 情境的模擬系統。
即使您不懂程式碼，也可以透過內建的「測試面板」來控制系統的行為，體驗開發者如何除錯與測試。

---

## 🚀 快速開始

1.  確認系統已啟動（若您看到此畫面，代表已經成功啟動）。
2.  開啟瀏覽器，網址通常為 `http://localhost:5173`。

---

## 🧪 核心功能：測試面板 (Dev Panel)

這是本系統最神奇的地方！請看畫面 **右下角** 有一個 **🧪 (試管圖示)** 按鈕。
點擊它，會展開一個控制面板，讓您「扮演上帝」，決定系統接下來會發生什麼事。

### 面板功能介紹

| 設定項目 | 功能說明 | 常用情境 |
| :--- | :--- | :--- |
| **Login Response** | 決定「登入時」後端會回傳什麼結果。 | 測試密碼輸入錯誤、帳號不存在、或是伺服器壞掉時的畫面反應。 |
| **User Role** | 決定登入後的「身分權限」。 | 切換 **Admin (管理員)** 或 **User (一般用戶)** 來測試不同權限能看到的頁面。 |
| **API Delay** | 模擬網路速度慢的狀況。 | 設定延遲 (如 1s)，可以觀察按鈕的「載入中 (Loading)」轉圈圈動畫。 |
| **API Status** | 決定登入後，取得資料 (如個資、列表) 的結果。 | 測試 Token 過期被自動登出，或資料載入失敗的狀況。 |
| **🗑️ 清除 Token** | 強制登出並清除所有紀錄。 | 當您想從頭開始測試，或測試「未登入」狀態時使用。 |

## 🎬 實戰演練：您可以跟著這樣做

建議您依照以下順序，體驗不同的測試情境：

### 1️⃣ 情境一：順利登入 (Happy Path)
這是最標準的流程，模擬一切順利的狀況。

1.  打開 **測試面板 🧪**。
2.  將 `Login Response` 設定為 **Success (成功)**。
3.  將 `User Role` 設定為 **Admin**。
4.  在登入畫面輸入：
    *   **Email**: `test@example.com` (或任意符合格式的 Email)
    *   **密碼**: `password123` (需 8 碼以上，包含英文與數字)
5.  點擊「登入」。
6.  🎉 **結果**：您應該會順利進入 Dashboard，並看到歡迎訊息。

### 2️⃣ 情境二：登入失敗 (Error Handling)
測試當輸入錯誤或系統故障時，畫面是否正確提示。

1.  先點擊面板的 **🗑️ 清除 Token** 登出。
2.  打開面板，將 `Login Response` 改為 **Invalid Password (密碼錯誤)**。
3.  輸入任意帳號密碼登入。
4.  🛑 **結果**：畫面應出現紅色的「密碼錯誤」提示框，且無法進入。
5.  *試試看其他選項：切換成 `Server Error`，會看到「伺服器錯誤」的提示。*

### 3️⃣ 情境三：權限控管 (Admin vs User)
測試不同身分是否被確實限制。

1.  先登出。
2.  打開面板，將 `User Role` 改為 **User (一般用戶)**，並確保 `Login Response` 是 Success。
3.  登入系統。
4.  在上方選單找到 **Admin (後台)** 按鈕，點擊它。
5.  🚫 **結果**：系統會把您踢回 Dashboard，因為一般用戶沒有權限進入後台。
    *(若您這時改回 Admin 權限並重新登入，就能順利進入了！)*

### 4️⃣ 情境四：體驗網路塞車 (Loading State)
1.  打開面板，將 `API Delay` 設定為 **1s (1秒)** 或更長。
2.  嘗試登入。
3.  ⏳ **結果**：您會明顯看到登入按鈕上的「轉圈圈」動畫持續了一段時間才進入，這確認了系統有正確處理等待狀態。

### 5️⃣ 情境五：如果 Token 過期了？ (Session Expiry)
模擬您登入很久沒操作，或是憑證失效的狀況。

1.  先成功登入並停留在 Dashboard。
2.  打開面板，將 `API Status (GET /api/me)` 設定為 **Token Expired**。
3.  重新整理網頁 (F5)。
4.  👋 **結果**：系統偵測到過期，會自動把您踢回登入頁，並告訴您「登入已過期」。

## 💡 常見問題

*   **Q: 為什麼我不管輸入什麼密碼都能登入？**
    *   A: 因為這是測試系統！只要面板設定為 `Success`，後端就會無條件讓您通過，方便您專注於測試前端流程。但不符合格式的 Email 或太短的密碼還是會被前端擋下來喔。
    
*   **Q: 我改了面板設定，為什麼沒反應？**
    *   A: 大部分設定會在「下一次呼叫 API」時生效。例如改了登入結果，要再按一次登入按鈕；改了 API 狀態，可能需要重新整理頁面才會觸發。
